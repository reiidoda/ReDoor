# Support

## Where to Ask for Help

- Usage questions and setup help: open a GitHub issue.
- Bug reports: use the bug issue template.
- Feature proposals: use the feature request template.
- Security vulnerabilities: follow `SECURITY.md` (private disclosure only).

## Issue Routing

When opening an issue, include:
- affected component (`client`, `RedoorApp`, `relay-node`, `blockchain-node`, `directory-dht`),
- expected behavior,
- actual behavior,
- reproduction steps,
- relevant logs (redacted).

## Maintainer Expectations

Maintainers will triage based on:
- severity (security/reliability first),
- reproducibility,
- roadmap alignment.

For contribution workflow, see `CONTRIBUTING.md`.
