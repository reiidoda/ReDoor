# Governance

## Project Model

Redoor is currently maintainer-led.

Maintainers are responsible for:
- roadmap prioritization,
- security and reliability gatekeeping,
- release decisions,
- moderation and community standards enforcement.

## Decision Process

- Normal changes are discussed in GitHub issues and reviewed in PRs.
- Security-sensitive changes require explicit threat/risk context and test evidence.
- Major cross-component changes should be tracked with a scoped issue and linked docs updates.

## Source of Truth

- Open work board: `docs/OPEN_SOURCE_STATUS.md`
- Contribution process: `CONTRIBUTING.md`
- Conduct standards: `CODE_OF_CONDUCT.md`
- Security reporting: `SECURITY.md`

## Change Policy

A change is considered complete when:
- code and tests are merged,
- relevant docs are updated,
- impacted gates/checks remain green in CI policy.
